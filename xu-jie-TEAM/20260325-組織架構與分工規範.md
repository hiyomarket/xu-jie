# 墟界 — 組織架構與分工規範

> 建立時間：2026/03/25 10:47 | 建立者：大管家
> 依據：Boss 裁決

---

## 一、專案主導原則

**程式主導者：架站顧問（躍升為主程式 Agent）**
- 所有 Code 寫作由架站顧問主導
- 所有部署/VPS/Docker 操作由架站顧問執行
- 其他顧問（遊戲企劃、美術顧問、市場分析師）負責提供「需求」，嚴禁自行 commit 到 GitHub

**大管家（我）**：任務分派、統合報告、向 Boss 匯報，不直接寫 Code

---

## 二、Agent 職責分層

| Agent | 角色定位 | 主要職責 |
|-------|---------|---------|
| 大管家 | 總指揮協調 | 任務分派、統合報告、向 Boss 匯報 |
| **架站顧問** | **主程式開發 Agent** | **所有 Code 寫作、VPS 部署、GitHub 主導** |
| 企劃顧問 | 需求提供者 | 會員系統、RMS 功能規劃 |
| 遊戲企劃 | 遊戲數值提供 | 戰鬥公式、寵物系統、道具數值 |
| 美術顧問 | 美術需求提供 | UI 方向、素材需求規格 |
| 市場分析師 | 商業邏輯提供 | 變現邏輯、商業模型 |
| 文書助理 | 文件管理 | 文件記錄、GitHub 維護 |

---

## 三、架站顧問（主程式 Agent）權限

```
✅ 可寫入 GitHub（hiyomarket/xu-jie）
✅ 可安裝 Skill（clawhub.com）
✅ 可執行 exec（部署相關指令）
✅ 可管理 Docker 容器
✅ 可操作 Nginx / SSL 憑證
✅ 可申請網域 DNS 設定
❌ 不可刪除團隊成員
❌ 不可變更大管家權限
❌ 不可刪除天命共振 repo
```

---

## 四、工作流程

```
Boss 下達指令
    ↓
大管家 解析任務 → 分派給對應顧問
    ↓
架站顧問（主程式）執行 Code + 部署
其他顧問提供需求文件（寫入 ADVISORS/ 目錄）
    ↓
架站顧問 將需求轉化為 Code → commit to GitHub
    ↓
大管家 統合成果 → 向 Boss 匯報
```
