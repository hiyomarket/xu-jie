## 1. 設計原則

| 原則 | 說明 |
|------|------|
| **無職業限制** | 玩家不需選擇職業，所有技能開放學習 |
| **任務解鎖** | 技能需透過 NPC 任務挑戰後才能學習 |
| **元素門檻** | 五大元素數值（火/水/風/地/金）達標才能學習對應屬性的高階技能 |
| **技能等級** | 每項技能可升級 Lv.1-10，影響威力與效果 |
| **技能欄位** | 玩家最多同時裝備 8 個技能（可替換） |

## 3. 學習機制

### 3.1 學習流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   發現 NPC   │───▶│  挑戰任務   │───▶│  滿足條件   │───▶│  習得技能   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
     │                  │                  │                  │
     ▼                  ▼                  ▼                  ▼
  • 城市NPC         • 試煉BOSS         • 等級門檻          • 扣除魔幣
  • 隱藏NPC         • 謎題解鎖         • 元素數值          • 佔用技能欄
  • 野外導師         • 收集素材         • 任務進度          • 習得成功
```

### 3.2 學習條件

| 條件類型 | 說明 | 範例 |
|----------|------|------|
| **等級門檻** | 玩家等級需達標 | 火球術需 Lv.10 |
| **元素數值** | 對應元素屬性需達標 | 烈焰風暴需火屬性 50 |
| **任務進度** | 需完成前置任務 | 需完成「魔法師的傳承」 |
| **魔幣費用** | 學習費用（可設定為 0） | 一般技能 100 魔幣 |
| **技能欄位** | 需有空位才能學習 | 最多 8 格 |

### 3.3 技能等級機制

| 等級 | 累積經驗 | 效果加成 |
|------|----------|----------|
| Lv.1 | 0        | 基礎效果 |
| Lv.2 | 100      | +10%     |
| Lv.3 | 250      | +20%     |
| Lv.4 | 500      | +30%     |
| Lv.5 | 1000     | +40%     |
| Lv.6 | 2000     | +50%     |
| Lv.7 | 4000     | +60%     |
| Lv.8 | 8000     | +70%     |
| Lv.9 | 16000    | +80%     |
| Lv.10| 32000    | +100%    |

## 6. 資料庫設計

### 6.1 技能目錄表 (game_skill_catalog)

現有欄位（遷移自天命共振）：
- `skill_id`, `name`, `wuxing`, `category`, `tier`
- `mp_cost`, `description`, `skill_type`, `is_active`
- `rarity`, `cooldown`, `power_percent`, `learn_level`

**新增欄位**：
```sql
ALTER TABLE game_skill_catalog ADD COLUMN acquire_type ENUM('shop', 'task', 'event', 'drop') DEFAULT 'shop';
ALTER TABLE game_skill_catalog ADD COLUMN task_id INT;           -- 綁定任務ID
ALTER TABLE game_skill_catalog ADD COLUMN element_requirement JSON; -- 元素門檻 {"fire": 30, "water": 0}
ALTER TABLE game_skill_catalog ADD COLUMN skill_rank INT DEFAULT 1; -- 技能階級 (1-3)
ALTER TABLE game_skill_catalog ADD COLUMN skill_exp_curve JSON;   -- 升級經驗曲線
```

### 6.2 玩家技能表 (user_skills)

```sql
CREATE TABLE user_skills (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL,
  skill_id VARCHAR(20) NOT NULL,
  skill_level INT DEFAULT 1,
  skill_exp INT DEFAULT 0,
  is_equipped BOOLEAN DEFAULT FALSE,
  equipped_slot INT,  -- 1-8 技能欄位
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uk_user_skill (user_id, skill_id)
);
```

### 6.3 任務表 (tasks)

```sql
CREATE TABLE tasks (
  id INT PRIMARY KEY AUTO_INCREMENT,
  task_code VARCHAR(50) UNIQUE NOT NULL,
  task_name VARCHAR(100) NOT NULL,
  description TEXT,
  task_type ENUM('main', 'side', 'daily', 'event', 'trial', 'awaken') DEFAULT 'main',
  prerequisite_task_code VARCHAR(50),  -- 前置任務代碼
  requirement_level INT,               -- 等級需求
  requirement_items JSON,              -- 需要物品 [{"item_id": "fire_feather", "count": 1}]
  requirement_monsters JSON,           -- 需要擊殺怪物 [{"monster_id": "M_FIRE_001", "count": 3}]
  npc_id INT,                          -- 交付NPC
  reward_skill_id VARCHAR(20),         -- 獎勵技能ID
  reward_item_json JSON,              -- 獎勵物品
  reward_currency INT DEFAULT 0,       -- 獎勵魔幣
  reward_element JSON,                 -- 獎勵元素加成 {"fire": 10, "water": 0}
  is_repeatable BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 6.4 玩家任務進度表 (user_tasks)

```sql
CREATE TABLE user_tasks (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL,
  task_code VARCHAR(50) NOT NULL,
  task_status ENUM('not_started', 'in_progress', 'completed', 'failed') DEFAULT 'not_started',
  progress_json JSON,                  -- 任務進度 {"kill_count": 3, "items_collected": 1}
  completed_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uk_user_task (user_id, task_code)
);
```

## 9. 風險與對策

| 風險 | 說明 | 對策 |
|------|------|------|
| 技能過多導致平衡問題 | 109 個技能難以全部測試 | MVP 先上 30 個核心技能 |
| 元素數值計算複雜 | 五行相剋與技能加成 | 設計簡化版數值表 |
| 任務設計成本高 | 每個技能對應一個任務 | 設計任務模板，快速生成 |
| 玩家不知道怎麼學習 | 沒有明確指引 | UI 提示 + 任務追蹤系統 |
| BOSS 戰鬥難度失衡 | 試煉BOSS 需要合理難度 | 根據玩家戰力動態調整 |
