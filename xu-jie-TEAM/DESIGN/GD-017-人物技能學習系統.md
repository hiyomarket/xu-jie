## 1. 概述

本設計書參考魔力寶貝的技能學習機制，為墟界設計一套「**任務解鎖 + 元素條件**」的技能學習系統。不同於魔力寶貝的職業綁定，墟界採用**無職業設計**，改以任務門檻和元素條件作為學習前提，讓玩家透過挑戰和解謎來獲得強大的技能。

---

## 2. 設計原則

| 原則 | 說明 |
|------|------|
| **無職業限制** | 玩家不需選擇職業，所有技能開放學習 |
| **任務解鎖** | 技能需透過 NPC 任務挑戰後才能學習 |
| **元素門檻** | 五大元素數值達標才能學習對應屬性的高階技能 |
| **技能等級** | 每項技能可升級，影響威力與效果 |
| **技能欄位** | 玩家最多同時裝備 8 個技能（可替換） |

## 4. 技能學習機制

### 4.1 學習流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   發現 NPC   │───▶│  挑戰任務   │───▶│  滿足條件   │───▶│  習得技能   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
     │                  │                  │                  │
     ▼                  ▼                  ▼                  ▼
  • 城市NPC         • 試煉BOSS         • 等級門檻          • 扣除魔幣
  • 隱藏NPC         • 謎題解鎖         • 元素數值          • 佔用技能欄
  • 野外導師         • 收集素材         • 任務進度          • 習得成功
```

### 4.2 學習條件

| 條件類型 | 說明 | 範例 |
|----------|------|------|
| **等級門檻** | 玩家等級需達標 | 火球術需 Lv.10 |
| **元素數值** | 對應元素屬性需達標 | 烈焰风暴需火属性 50 |
| **任務進度** | 需完成前置任務 | 需完成「魔法師的傳承」 |
| **魔幣費用** | 學習費用（可設定為 0） | 一般技能 100 魔幣 |
| **技能欄位** | 需有空位才能學習 | 最多 8 格 |

### 4.3 技能等級機制

```
技能等級與經驗需求：

| 等級 | 累積經驗 | 效果加成 |
|------|----------|----------|
| Lv.1 | 0        | 基礎效果 |
| Lv.2 | 100      | +10%     |
| Lv.3 | 250      | +20%     |
| Lv.4 | 500      | +30%     |
| Lv.5 | 1000     | +40%     |
| Lv.6 | 2000     | +50%     |
| Lv.7 | 4000     | +60%     |
| Lv.8 | 8000     | +70%     |
| Lv.9 | 16000    | +80%     |
| Lv.10| 32000    | +100%    |

經驗來源：戰鬥中使用技能、任務、特定的修煉NPC
```

## 7. 資料庫欄位設計

### 7.1 技能目錄表 (game_skill_catalog)

現有欄位（已有）：
- `skill_id`, `name`, `wuxing`, `category`, `tier`
- `mp_cost`, `description`, `skill_type`, `is_active`
- `rarity`, `cooldown`, `power_percent`, `learn_level`

**需新增欄位**：
```sql
ALTER TABLE game_skill_catalog ADD COLUMN acquire_type ENUM('shop', 'task', 'event', 'drop') DEFAULT 'shop';
ALTER TABLE game_skill_catalog ADD COLUMN task_id INT;           -- 綁定任務ID
ALTER TABLE game_skill_catalog ADD COLUMN element_requirement JSON; -- 元素門檻 {"fire": 30, "water": 0}
ALTER TABLE game_skill_catalog ADD COLUMN skill_rank INT DEFAULT 1; -- 技能階級 (1-3)
ALTER TABLE game_skill_catalog ADD COLUMN skill_exp_curve JSON;   -- 升級經驗曲線
```

### 7.2 玩家技能表 (user_skills)

```sql
CREATE TABLE user_skills (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL,
  skill_id VARCHAR(20) NOT NULL,
  skill_level INT DEFAULT 1,
  skill_exp INT DEFAULT 0,
  is_equipped BOOLEAN DEFAULT FALSE,
  equipped_slot INT,  -- 1-8 技能欄位
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 7.3 任務表 (tasks)

```sql
CREATE TABLE tasks (
  id INT PRIMARY KEY AUTO_INCREMENT,
  task_code VARCHAR(50) UNIQUE,
  task_name VARCHAR(100),
  description TEXT,
  task_type ENUM('main', 'side', 'daily', 'event'),
  prerequisite_task_id INT,  -- 前置任務
  requirement_json JSON,    -- 任務條件 {"level": 10, "items": ["fire_feather"]}
  reward_skill_id VARCHAR(20),  -- 獎勵技能ID
  reward_item_json JSON,
  reward_currency INT,
  is_repeatable BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 10. 風險與對策

| 風險 | 說明 | 對策 |
|------|------|------|
| 技能過多導致平衡問題 | 109 個技能難以全部測試 | MVP 先上 30 個核心技能 |
| 元素數值計算複雜 | 五行相剋與技能加成 | 設計簡化版數值表 |
| 任務設計成本高 | 每個技能對應一個任務 | 設計任務模板，快速生成 |
| 玩家不知道怎麼學習 | 沒有明確指引 | UI 提示 + 任務追蹤系統 |
