## 1. 概述

本設計書定義墟界的完整戰鬥系統，包含三種戰鬥模式：掛機模式、玩家模式、PVP/BOSS戰鬥模式。所有戰鬥均參考魔力寶貝的回合制玩法，採用文字化敘述呈現。

## 3. 模式一：掛機模式

### 3.1 核心機制

```
┌─────────────────────────────────────────────────────────┐
│                    掛機模式流程                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  開始掛機 → 探索(15秒) → 隨機事件 → 結算 → 循環        │
│       │              │              │                   │
│       ▼              ▼              ▼                   │
│    探索中         戰鬥/採集      獎勵結算               │
│    (隨機)        沒體力→注靈     (1/3獎勵)             │
│                   沒MP→休息                          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 3.2 隨機事件權重

| 事件類型 | 權重 | 說明 |
|----------|------|------|
| 遇敵戰鬥 | 40% | 隨機怪物戰鬥 |
| 採集資源 | 30% | 獲得素材 |
| 發現寶箱 | 10% | 隨機道具 |
| 無事件 | 20% | 空手而回 |

### 3.3 資源管理邏輯

```
掛機資源循環：

體力 (Energy)
    │
    ▼
體力 > 0 → 執行行動 → 扣 1 體力
    │
    ▼
體力 = 0 → 自動「注靈」→ 恢復 10 體力（消耗魔幣）
    │
    ▼
MP (魔力)
    │
    ▼
MP > 0 → 執行魔法 → 扣 MP
    │
    ▼
MP = 0 → 自動「休息」→ 恢復 20% 最大 MP
    │
    ▼
MP 恢復 → 回到探索
```

### 3.4 獎勵計算

| 項目 | 掛機模式 | 說明 |
|------|----------|------|
| 經驗值 | ×0.33 | 只有原本的 1/3 |
| 金錢 | ×0.33 | 只有原本的 1/3 |
| 掉落率 | ×0.33 | 只有原本的 1/3 |
| 體力消耗 | 0 | 不扣體力 |
| MP消耗 | 正常 | 使用技能仍扣 MP |

## 5. 模式三：戰鬥視窗模式（PVP/BOSS/地圖怪）

### 5.1 通用戰鬥框架

```
┌─────────────────────────────────────────────────────────┐
│                    回合制戰鬥框架                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  【先手判定】                                           │
│  敏捷最高者 → 先手                                      │
│  攻擊順序：人 → 寵 → 敵                                  │
│  （如有複数相同則隨機）                                  │
│                                                         │
│  【回合限制】                                           │
│  每回合 20 秒思考時間                                   │
│  超過 20 秒 → AI 自動下指令                             │
│                                                         │
│  【戰鬥單位】                                           │
│  玩家 + 寵物 = 2 單位                                   │
│  敵人 + 敵寵 = 2 單位（BOSS 除外）                      │
│                                                         │
│  【勝利條件】                                           │
│  敵方人寵均陣亡                                         │
│                                                         │
│  【失敗條件】                                           │
│  我方人寵均陣亡                                         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 5.2 可執行指令

| 指令 | 說明 | 冷卻 |
|------|------|------|
| ⚔️ 一般攻擊 | 對敵方單體造成物理傷害 | 0 |
| ✨ 技能使用 | 使用已裝備技能（消耗 MP） | 技能冷卻 |
| 🛡️ 防禦 | 本回合受到的傷害 -50% | 0 |
| 💊 道具使用 | 使用背包中的道具 | 0 |
| 🏳️ 投降 | 立即認輸（PvP 限定） | - |
| 🏃 逃跑 | 脫離戰鬥（20% 失敗率） | 0 |

### 5.3 戰鬥流程範例

```
回合 1：戰鬥開始
═══════════════════════════════

先手順序：
1. 葉刃狐（敏 25）
2. 玩家（敏 18）
3. 綠芽史萊姆（敏 12）
4. 敵寵（敏 10）

---

回合 1 - 葉刃狐行動：
> 葉刃狐使用「藤蔓束縛」！
> 玩家被束縛，無法行動 1 回合！

---

回合 1 - 玩家行動（被束縛）：
> 玩家處於束縛狀態，無法行動！
> 系統自動進行防禦！

---

回合 1 - 綠芽史萊姆行動：
> 綠芽史萊姆使用「撞擊」攻擊葉刃狐！
> 造成 35 點傷害！
> 葉刃狐 HP: 120 → 85

---

回合 1 - 敵寵行動：
> 敵寵使用「抓撓」攻擊綠芽史萊姆！
> 造成 20 點傷害！
> 綠芽史萊姆 HP: 100 → 80

---

回合結束結算：
═══════════════════════════════
玩家：HP 150/150（束縛中，下回合無法行動）
綠芽史萊姆：HP 80/100
葉刃狐：HP 85/120
敵寵：HP 90/90
敵人：HP 120/120

回合 2 倒數：20 秒
```

## 7. AI 自動戰鬥邏輯

### 7.1 超時AI行為

```
當玩家超過 20 秒未下指令時，系統自動執行：

優先順序：
1. 有 MP → 使用威力最高的技能
2. 無 MP → 一般攻擊
3. HP < 30% → 使用道具（如果有的話）
4. 上一回合被攻擊 → 防禦
```

### 7.2 AI 決策樹

```
AI 決策邏輯：
           │
           ▼
    HP > 50%？
    ├── 是 → 攻擊或使用技能
    └── 否 → 
              │
              ▼
         有恢復道具？
         ├── 是 → 使用恢復
         └── 否 →
                   │
                   ▼
              有吸血技能？
              ├── 是 → 使用吸血
              └── 否 → 防禦
```

## 9. 資料庫設計

### 9.1 戰鬥記錄表 (battle_logs)

```sql
CREATE TABLE battle_logs (
  id INT PRIMARY KEY AUTO_INCREMENT,
  battle_type ENUM('pve', 'pvp', 'world_boss', 'dungeon') NOT NULL,
  user_id INT NOT NULL,
  monster_id INT,
  result ENUM('win', 'lose', 'draw', 'fled') NOT NULL,
  duration_seconds INT,
  exp_gained INT,
  gold_gained INT,
  items_dropped JSON,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 9.2 戰鬥回合記錄表 (battle_rounds)

```sql
CREATE TABLE battle_rounds (
  id INT PRIMARY KEY AUTO_INCREMENT,
  battle_id INT NOT NULL,
  round_number INT NOT NULL,
  attacker_type ENUM('player', 'pet', 'monster') NOT NULL,
  attacker_id INT NOT NULL,
  defender_type ENUM('player', 'pet', 'monster') NOT NULL,
  defender_id INT NOT NULL,
  action_type ENUM('attack', 'skill', 'defend', 'item', 'flee') NOT NULL,
  skill_id VARCHAR(20),
  damage_dealt INT,
  damage_taken INT,
  hp_after_action INT,
  mp_after_action INT,
  action_result JSON,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 11. 風險與對策

| 風險 | 說明 | 對策 |
|------|------|------|
| 掛機獎勵過低 | 1/3 獎勵可能讓玩家不想掛機 | 提供掛機專屬福利 |
| PvP 掛機問題 | 20 秒太長可能被利用 | 實裝掛機檢測 |
| BOSS 排程 | 同時只能一組挑戰 | 設計排隊系統 |
| 網路延遲 | 回合制需要穩定網路 | 設計離線緩存 |
