## 二、三種執行模式

### 2.1 掛機模式（所有玩家適用）

**觸發**：玩家按下「⚔ 開始掛機」按鈕

**每15秒自動執行一次事件輪迴：**
```
探索 → (隨機判定) → 戰鬥 或 採集 → 結算 → 循環
  ↑                                          │
  └── 無體力→注靈(耗魔幣) / 無MP→休息 ──┘
```

**隨機事件權重：**

| 事件 | 機率 | 說明 |
|------|------|------|
| 遇敵戰鬥 | 40% | 隨機怪物，快速結算 |
| 採集資源 | 30% | 獲得素材 |
| 發現寶箱 | 10% | 隨機道具 |
| 無事件 | 20% | 空手而回 |

**獎勵倍率：**

| 項目 | 倍率 | 說明 |
|------|------|------|
| 經驗值 | ×0.33 | 只有原本的 1/3 |
| 金錢 | ×0.33 | 只有原本的 1/3 |
| 掉落率 | ×0.33 | 只有原本的 1/3 |
| 體力消耗 | **0** | 不扣體力 |
| MP消耗 | 正常 | 使用技能仍扣 MP |

**戰鬥結算畫面（普通怪，不阻擋）：**
```
🟢 掛機戰鬥結算
────────────────────────────────
你　 VS  迷失狼（Lv.3）
───────────────────────────
你的 HP： ████████░░  80/100
狼　 HP： ░░░░░░░░░░  0/100
───────────────────────────
✅ 勝利！
🎁 獲得：經驗+45 ｜ 金幣+12 ｜ 獸肉×1
────────────────────────────────
⏱ 15秒後自動繼續...
```

**精英怪通知（精英/首領級）：**
右上角彈出通知：`⚠ 精英怪遭遇！[查看]` 可選擇快速結算或進入戰鬥視窗

**離線掛機邏輯：**
- 離線時長 → 掛機回合數（每15秒=1回合）
- 最大離線掛機時長：**8小時（封頂1920回合）**
- 玩家重新登入 → 顯示「📦 離線掛機報告」彈窗
  - 總戰鬥次數 / 勝敗 / 總經驗 / 總金幣 / 掉落物品

## 三、三種視窗戰鬥模式

### 3.1 通用回合制框架

所有視窗戰鬥（ PVP / 地圖行動怪 / Boss戰 ）共用以下規則：

**行動單位：**
- 玩家 + 戰鬥寵 = 2個行動單位（分開排序）

**速度結算：**
- 所有存活行動單位按 **SPD 降冪排序**
- 同速時：玩家角色優先於寵物；不同玩家同速時用時間戳+UUID雜湊隨機
- 公式：`final_speed = spd * 1000 + (500 if CHARACTER else 0) + random_hash % 500`

**每回合 20 秒思考時間：**
- 前端顯示倒數計時器（倒數到0自動觸發AI）
- 後端用 Redis SETEX 鍵值過期防作弊（`battle:turn:{battle_id}:{unit_id}` TTL=20）
- 超時自動執行AI決策（見AI章節）

**指令格式（JSON）：**
```json
{
  "command_type": "ATTACK | SKILL | ITEM | DEFEND | SURRENDER | FLEE",
  "battle_id": "uuid",
  "unit_id": "uuid",
  "turn_number": 3,
  "payload": { "target_id": "unit_uuid" },
  "timestamp": 1743000000,
  "client_hash": "md5(...)
}
```

---

### 3.2 PVP 戰鬥

**參與條件：** 雙方玩家同意對戰，等級差 ≤ 10級

**戰鬥規則：**
- 每回合 20 秒思考時間
- 人寵同上陣（2v2）
- **投降**：立即結束戰鬥，判定為敗
- **逃跑**：20%失敗率，失敗則浪費一回合指令
- **勝利條件**：一方人寵均陣亡

**獎勵與懲罰：**
- 勝利：榮譽點數 + PvP貨幣 + 連勝加成
- 失敗：少量安慰獎
- 死亡無懲罰（寵物不會死亡）

**對手離線處理：**
- 離線30秒內：等待計時
- 離線30秒後：AI接管
- 離線120秒後：從未操作 → 直接判負

---

### 3.3 地圖行動怪（世界首領）

**怪物特性：**
- 結點移動的小首領怪，全服玩家共用
- 後台設定復活時間（預設30分鐘）
- 死亡後進入復活冷卻，復活後全服廣播通知

**觸發條件：**
- 掛機模式中遭遇 → 彈出選擇：快速結算 OR 進入戰鬥視窗
- 玩家手動遭遇 → 直接進入戰鬥視窗
- 漫遊模式：只在其周圍1格有玩家判定才主動進攻

**逃跑機制：**
- 逃跑：20%失敗率，失敗則浪費一回合指令
- 逃跑成功：立即脫離戰鬥

**勝利條件：** 人寵均存活且魔物HP歸零
**敗北條件：** 人寵均陣亡

**獎勵：**
- 首殺獎勵豐厚（MVP制度：對魔物造成最高傷害者額外加成）
- 參與獎勵（所有對魔物造成傷害的玩家）

---

### 3.4 Boss戰 / 惡魔城挑戰

**參與限制：**
- **全服每次只能有一組挑戰**（個人或組隊）
- 挑戰中顯示「⚔ 挑戰中，不可申請」鎖定狀態
- 需開發組隊模式（見Phase 3）

**戰鬥規則：**
- 每回合 20 秒思考時間
- 人寵同上陣
- **不可逃跑**
- **不可投降**（Boss戰為持續挑戰）
- 敏高者先手

**Boss特色：**
- 多階段形態（HP 66%/33% 各觸發一次階段轉換）
- 召喚小怪
- 特殊技能

**勝利條件：** Boss HP 歸零
**敗北條件：** 玩家與寵物均陣亡

**獎勵：**
- 首殺獎勵（豐厚）
- 稀有道具 / 寵物
- 成就稱號
- 傳說掉落表

## 四、戰鬥流程狀態機

### 4.1 狀態列舉

```
enum BattleState {
  START,           // 戰鬥初始化
  SPEED_SORT,     // 速度排序
  TURN_BEGIN,     // 回合開始
  PLAYER_TURN,    // 玩家指令輸入
  ENEMY_TURN,     // 敵方AI指令
  CALCULATE,       // 行動結算
  STATUS_EFFECT,   // 狀態效果處理
  CHECK_END,       // 終結條件檢查
  END,             // 戰鬥結束
  INTERRUPTED      // 中斷（投降/逃跑成功/超時）
}
```

### 4.2 狀態流轉圖

```
[START]
    │
    ▼
[SPEED_SORT] ────────────────────────────────
    │                                         
    ▼                                         
[TURN_BEGIN] ◄────────────────────────────┐
    │                                         
    ├──→ [PLAYER_TURN] ─→ [ENEMY_TURN] ─┤
    │                       ▲               │
    └──→ (掛機快速結算跳過) ─┘               │
              ▼                               │
        [CALCULATE]                           │
              ▼                               │
        [STATUS_EFFECT]                       │
              ▼                               │
        [CHECK_END] ──未結束──→ [TURN_BEGIN] ─┘
              │                                         
         已結束 ▼                                 
            [END] ◄─── (投降/逃跑/人寵均陣亡)
```

### 4.3 各狀態行為

| 狀態 | 核心行為 | 計時 |
|------|---------|------|
| START | 載入雙方屬性、寵物、技能，鎖定挑戰狀態 | — |
| SPEED_SORT | 收集所有單位SPD，降冪排列 | — |
| TURN_BEGIN | 回合計數+1、刷新狀態、寫入戰鬥日誌 | — |
| PLAYER_TURN | 等待玩家提交指令 | **20秒**，超時→AI接管 |
| ENEMY_TURN | AI決策（隨機500-1500ms模擬思考）| — |
| CALCULATE | 物理傷害：`ATK×2 - DEF÷2` ×隨機係數 | — |
| STATUS_EFFECT | DOT/HOT結算、狀態回合-1 | — |
| CHECK_END | 判斷勝敗條件（見各模式）| — |
| END | 結算獎勵、釋放鎖、寫入日誌 | — |
| INTERRUPTED | 標記中斷、釋放資源 | — |

| 💊 道具使用 | 使用背包中的道具 | 0 | — |
| 🏳️ 投降 | PVP立即認輸（其餘模式不適用） | — | PVP限定 |
| 🏃 逃跑 | 脫離戰鬥（20%失敗率，失敗浪費一回合）| 0 | 地圖怪/Boss限定 |

### 5.2 指令驗證優先順序

```
1. 檢查指令格式（JSON schema）
2. 檢查 battle_id 是否匹配
3. 檢查 turn_number 是否為當前回合
4. 檢查 unit_id 是否為己方存活單位
5. 檢查 MP 是否足夠（技能）
6. 檢查冷卻是否結束
7. 檢查道具是否在背包中
8. 執行指令
```

**無效指令錯誤碼：**
- `ERR_INVALID_TARGET`：目標無效或已倒下
- `ERR_NO_MP`：MP不足
- `ERR_COOLDOWN`：技能冷卻中
- `ERR_NO_ITEM`：道具不在背包
- `ERR_WRONG_TURN`：不是你的行動回合
- `ERR_ALREADY_ACTED`：本回合已行動
- `ERR_BATTLE_ENDED`：戰鬥已結束

## 六、AI 自動決策

### 6.1 超時AI行為（20秒無操作時觸發）

```
AI決策優先順序：
1. HP > 50%？→ 使用威力最高的技能
2. HP < 30%？→ 使用道具（補HP）
3. 無MP？→ 一般攻擊
4. 上一回合被攻擊？→ 防禦
5. 其他 → 一般攻擊
```

### 6.2 AI決策樹

```
HP > 50%？
  ├── 是 → 有MP？
  │         ├── 是 → 使用威力最高技能（ATK/MAG最高）
  │         └── 否 → 一般攻擊
  └── 否 → HP < 30%？
              ├── 是 → 有補血道具？
              │         ├── 是 → 使用補血道具
              │         └── 否 → 防禦
              └── 否 → 有MP？
                        ├── 是 → 使用吸血/治療技能
                        └── 否 → 防禦
```

## 八、快速結算演算法（掛機遭遇行動怪）

**輸入：** 玩家戰力 vs 行動怪戰力

**傷害公式（魔力寶貝經典）：**
```
物理傷害 = (ATK × 2 - DEF ÷ 2) × 隨機係數(0.85~1.15)
魔法傷害 = (MAG × 2 - MDEF ÷ 2) × 隨機係數(0.85~1.15)
```

**勝率計算：**
```
勝率 = 玩家ATK / (玩家ATK + 怪物DEF)
戰力比上限：90%（永遠留10%失敗空間）
戰力比下限：10%（永遠有10%勝利機會）
```

**BP檔位加權：**
- S檔：×1.5（A檔×1.25 / B檔×1.0 / C檔×0.85 / D檔×0.70）

**結算流程：**
1. 根據勝率亂數決定勝負
2. 計算交換傷害（各方損失HP）
3. 勝利方HP歸零：失敗
4. 失敗方HP歸零：勝利
5. 每日50次上限，防止外掛刷幣

## 十、資料庫Schema

```sql
-- 戰鬥實例
CREATE TABLE game_battles (
  battle_id      VARCHAR(36) PRIMARY KEY,
  battle_type    ENUM('pve_idle','pve_manual','pvp','map_mob','boss') NOT NULL,
  realm         VARCHAR(20),          -- 初界/中界/高界
  p1_user_id    VARCHAR(36),
  p2_user_id    VARCHAR(36),          -- PVP時對手
  mob_instance_id VARCHAR(36),          -- 地圖行動怪/BOSS ID
  start_time    TIMESTAMP NOT NULL,
  end_time     TIMESTAMP,
  result       ENUM('win','lose','draw','fled','interrupted') NOT NULL,
  reward_exp   INT,
  reward_gold  INT,
  is_idle      BOOLEAN DEFAULT FALSE,  -- 掛機標記
  turn_count   INT DEFAULT 0,
  created_at   TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 戰鬥參與者快照（屬性在戰鬥開始時鎖定）
CREATE TABLE game_battle_participants (
  participant_id VARCHAR(36) PRIMARY KEY,
  battle_id     VARCHAR(36) NOT NULL,
  entity_type   ENUM('player','pet','monster') NOT NULL,
  entity_id    VARCHAR(36) NOT NULL,
  team         ENUM('ally','enemy') NOT NULL,
  hp           INT NOT NULL,
  max_hp       INT NOT NULL,
  mp           INT NOT NULL,
  max_mp       INT NOT NULL,
  atk           INT NOT NULL,
  def           INT NOT NULL,
  spd           INT NOT NULL,
  mag           INT NOT NULL,
  turn_order    INT NOT NULL,         -- 速度排序後的順序
  stance       VARCHAR(20) DEFAULT 'normal',  -- normal/defend/stunned等
  is_alive     BOOLEAN DEFAULT TRUE,
  INDEX idx_battle (battle_id)
);

-- Buff/狀態效果
CREATE TABLE game_battle_buffs (
  buff_id       VARCHAR(36) PRIMARY KEY,
  battle_id     VARCHAR(36) NOT NULL,
  participant_id VARCHAR(36) NOT NULL,
  buff_key      VARCHAR(50) NOT NULL,  -- poison/burn/stun等
  remaining_turns INT NOT NULL,
  extra_data   JSON,
  created_at    TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_participant (participant_id)
);

-- 掛機記錄主表
CREATE TABLE game_idle_sessions (
  session_id   VARCHAR(36) PRIMARY KEY,
  user_id      VARCHAR(36) NOT NULL,
  realm        VARCHAR(20) NOT NULL,
  start_time   TIMESTAMP NOT NULL,
  end_time     TIMESTAMP,
  total_ticks  INT DEFAULT 0,         -- 掛機總回合數
  FOREIGN KEY (user_id) REFERENCES users(user_id)
);

-- 掛機每回合記錄
CREATE TABLE game_idle_ticks (
  tick_id      BIGINT AUTO_INCREMENT PRIMARY KEY,
  session_id   VARCHAR(36) NOT NULL,
  tick_number  INT NOT NULL,
  event_type   ENUM('explore','battle','gather','chest','nothing') NOT NULL,
  monster_id   VARCHAR(36),
  result       ENUM('win','lose','draw','none') DEFAULT 'none',
  exp_gained   INT DEFAULT 0,
  gold_gained  INT DEFAULT 0,
  items_gained JSON,
  created_at   TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_session (session_id)
);

-- 排程指令（下回合預排）
CREATE TABLE game_battle_commands (
  command_id   VARCHAR(36) PRIMARY KEY,
  battle_id    VARCHAR(36) NOT NULL,
  unit_id      VARCHAR(36) NOT NULL,
  turn_number  INT NOT NULL,
  command_type ENUM('attack','skill','item','defend','surrender','flee') NOT NULL,
  target_id    VARCHAR(36),
  skill_id     VARCHAR(36),
  item_id      VARCHAR(36),
  is_pre_scheduled BOOLEAN DEFAULT FALSE,  -- 是否為提前排程
  submitted_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_battle_turn (battle_id, turn_number)
);

-- 戰鬥日誌流水
CREATE TABLE game_battle_logs (
  log_id      BIGINT AUTO_INCREMENT PRIMARY KEY,
  battle_id   VARCHAR(36) NOT NULL,
  turn_number INT,
  message     TEXT NOT NULL,         -- 完整文字敘述
  params      JSON,                  -- 結構化參數（用於重建）
  created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_battle (battle_id)
);

-- 行動怪實例（世界首領/地圖怪）
CREATE TABLE game_map_mobs (
  mob_instance_id VARCHAR(36) PRIMARY KEY,
  mob_template_id VARCHAR(36) NOT NULL,
  current_hp   INT NOT NULL,
  max_hp       INT NOT NULL,
  respawn_at    TIMESTAMP,           -- NULL=存活中
  current_node VARCHAR(20),          -- 當前結點座標
  move_interval INT DEFAULT 60,       -- 移動間隔（秒）
  last_move_at TIMESTAMP,
  is_alive    BOOLEAN DEFAULT TRUE
);
```
